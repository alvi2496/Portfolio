Use google compute engine as google colab backend

June 5, 2020

Alvi Mahadi

This blog illustrates how to create the google cloud compute engine, configure, install python in the instance and connect it with google colab front end.

Creating the instance

  1. Go to the cloud console dashboard.
  2. Create a new project if you do not have one.
  3. In the sidebar, you should see Compute Engine. Hover onto that and
  4. click on VM instances.
  5. Click on create an instance in the navbar. You should see a form for the creation of the instance. Fill the form to create the instance.
  6. Just under the Firewall option, expand the Management, security, networking, sole tenancy tab.
  7. Under the management tab, you will find the Metadata option. In the key box, enter enable-oslogin and value enter FALSE to make it unable for OS login. It will, in turn, make the ssh login possible for this instance.
  8. Now click create to create the instance. Once created, you will see this instance in the list of instances.
  9. Now click on the ssh button in the instance. It will pop up a terminal to access the instance.
  10. Add your ssh key to the default user's .ssh/authorized_keys so that you can ssh into it.
  11. You can check if you are able to connect to the instance using ssh by
$ ssh -i .ssh/id_rsa.pub username@ip_address

Install gcloud in local PC

  1. Please go to: https://cloud.google.com/sdk/docs/quickstarts to install cloud in your local machine.

Configure gcloud instance

  1. Initialize gcloud project using $ gcloud init. Follow the prompt to configure the gcloud sdk with your google account.
  2. Connect to the server and forward your port
gcloud compute ssh - zone ZONE_OF_THE_INSTANCE NAME_OF_THE_INSTANCE - -L 8081:127.0.0.1:8081

Replace ZONE_OF_THE_INSTANCE with the zone of your instance. You can find the zone under Zone tab. NAME_OF_THE_INSTANCE should be under Name tab. use 127.0.0.1 instead of localhost. Using localhost invoke a weird error message channel 3: open failed: administratively prohibited: open failed. I don't know why.

  1. Now you should be logged in to the compute engine.

Install, configure and run jupyter notebook

Install python

This demo installs python 3.6.10. Get the appropriate link for desired python version

  1. Install python 3.6.10
$ apt-get install build-essential
$ cd ~/Downloads && wget https://www.python.org/ftp/python/3.6.10/Python-3.6.10.tar.xz
$ cd Python-3.6.10
$ ./configure
$ make
$ make install
  1. Make a virtual python environment $ python3.6 -m venv venv
  2. Activate the virtual environment $ source venv/bin/activate

Install and configure jupyter notebook

$ pip install jupyter_http_over_ws
$ jupyter serverextension enable - py jupyter_http_over_ws

Run the notebook server

jupyter notebook \
  --NotebookApp.allow_origin='https://colab.research.google.com' \
  --port=8081 \
  --NotebookApp.port_retries=0 \
  --no-browser

Configure google colab to use the instance as backend

  1. Open google colab
  2. The top right corner just left to Editing, click on the down arrow beside Connect
  3. Click on Connect to local runtime
  4. In the url field, put the localhost address. You can find the address with the token as the output of the jupyter notebook server.
  5. Click connect and you should be good to go.
